# deepcode-audit

> AI 写的代码，让 AI 来审计——不是单独一个 AI，而是一队 AI 交叉验证。

## 是什么

deepcode-audit 是一个 [Claude Code](https://claude.ai/code) skill，对你的代码库逐文件、逐行审计，使用 **8 个核心视角 + N 个领域视角** 交叉验证。

灵感来自 MMO 任务板——每个文件 × 每个视角 = 一个任务。冒险者领取任务、逐文件阅读、执行七层审查、提交证据。公会会长验证每一条提交。

## 亮点

- **8 视角交叉验证**：工程 / 产品 / 质量 / AI 内容 / 自我审查 / 跨角色 / 安全合规 / 性能。同一文件被多个冒险者独立审查。
- **MMO 任务板调度**：公会会长 + 冒险者 + 任务领取/提交/验证/驳回循环，实时进度追踪。
- **防作弊机制**：冒险者必须引用每个文件的精确中间一行作为内容见证——不实际读文件就编不出来。
- **七层审查框架**：代码行 → 函数 → 文件 → 功能 → 模块 → 架构 → 系统，自底向上，禁止跳层。
- **自动修复 + 回归审计**：可安全自动修复的项直接修、提交、重审。需要人工判断的走交互式引导。
- **领域自动发现**：扫描源码时检测到数据工程、移动端等关键词，自动激活对应领域视角。

## 快速开始

1. 将 skill 加载到 Claude Code（放入 `.claude/skills/` 或通过 skill 管理加载）
2. 触发词：`全量审计` / `deep audit` / `系统审计` / `code audit` / `架构审计` / `深度审计` / `安全审计` / `合规审计`
3. Skill 扫描代码库、生成任务板、逐文件调度审查

## 配置

可选：编辑 `audit-config.yaml` 调整审计范围、启用/禁用视角、或调优调度参数。不配置则使用默认值。
